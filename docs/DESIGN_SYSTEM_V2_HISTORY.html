1.1 Critical Accessibility Fixes

Fixed 3 blocking accessibility violations that prevented WCAG 2.1 AA compliance:

Switch Component (Switch.tsx:68)

// Before (Incorrect) aria-checked={props.checked || props.defaultChecked || false} // After (Fixed) aria-checked={props.checked !== undefined ? props.checked : undefined}

Impact: Screen readers now correctly announce switch state without false positives.

Button Component (Button.tsx)

// Added accessibility attributes aria-busy={loading} aria-live={loading ? 'polite' : undefined}

Impact: Loading states are now announced to screen reader users.

Input Component (Input.tsx:160)

// Before (Not keyboard accessible) <button tabIndex={-1}>√ó</button> // After (Keyboard accessible) <button>√ó</button>

Impact: Clear button now works with keyboard navigation (Tab + Enter).

‚ö†Ô∏è Breaking Changes

Switch Component Structure

The Switch component now has an additional outer wrapper to support error and helper text.

Before (v1.x)

<div className="ff-switch-wrapper"> {/* switch and label */} </div>

After (v2.0.0)

<div className="ff-switch-field"> {/* NEW outer wrapper */} <div className="ff-switch-wrapper"> {/* switch and label */} </div> {/* error and helper messages */} {/* NEW */} </div>

Migration Required

  • Review custom CSS targeting .ff-switch-wrapper
  • Update selectors if you need to target the outer container
  • Test Switch component in your application

See UPGRADE_GUIDE.md for detailed migration instructions.

Non-Breaking Changes

All other changes are backward compatible:

  • ‚úÖ New props are optional (variant, helperText)
  • ‚úÖ CSS variables don't break existing hardcoded overrides
  • ‚úÖ All components maintain existing API
  • ‚úÖ Existing functionality preserved
